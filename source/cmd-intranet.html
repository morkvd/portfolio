CMD-Intranet redesign met flexbox

Voor het vak frontend heb ik de homepage van het intranet van CMD responsive gemaakt, waarbij ik onder andere gebruik heb gemaakt van flexbox en responsive images. We kregen de kale HTML aangeleverd en moesten deze — zonder gebruik te maken van classes en IDâ€™s — stijlen in de huistijl van de CMD.

Om erachter te komen wat er allemaal mogelijk is met flexbox heb ik ervoor gekozen om de layout helemaal om te gooien bij de transitie van mobile naar desktop. Op kleine schermen is de scrol-richting verticaal en op schermen van tablet formaat wordt de scrol-richting horizontaal.

Wat ik leuk vond aan het vak frontend was, dat we werden aangemoedigd om lekker te experimenteren met css. Door dingen te proberen en lekker te experimenteren ontdek je de mogelijkheden en limitaties van het web en krijg je een beter gevoel over wat wel en niet mogelijk is.

Nadat ik een aantal klein lo-fi schetsjes had gemaakt ben ik gelijk aan de slag gegaan met het stijlen van de html. Ik heb mobile-first gewerkt door te beginnen met het maken van de mobiele website en daarna pas de tablet versie en de desktop layout erbovenop te bouwen.

Ik heb tijdens het maken van deze opdracht geleerd hoe flexbox werkt en hoe je responsive afbeeldingen gebruikt. Ook heb ik geleerd hoe de specificiteit van css regels werkt en hoe je dit kan gebruiken bij het schrijven van css.
